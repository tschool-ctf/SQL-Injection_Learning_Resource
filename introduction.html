SQL Injection 介紹

什麼是 SQL？

每個網站、每個 App 背後，都有一個巨大的「資料庫」。這個資料庫裡面存放著各式各樣的資訊，像是帳號、密碼、商品資訊等等。

而要跟這類資料庫溝通，我們需要特別的語言，（常見的）叫做「SQL (Structured Query Language)」，也就是「結構化查詢語言」。網站程式會用 SQL 語句來和資料庫要求取得、修改、新增或刪除資料等，就像是「請你給我A資料表格中的第五格資料」、「請你刪除X資料表格中的第八十一格資料」。

什麼是 SQL Injection？

那麼，「SQL Injection」是什麼呢？

它是一種駭客攻擊手法，就是基於前面所講的「SQL」溝通語言。原本，程式只讓你在特定欄位輸入資料或是要求的句子。但是，如果程式在處理這些輸入資料時不夠小心，沒有好好檢查輸入的內容，駭客就有機會利用這個漏洞，偷偷地在你的輸入內容中，夾帶一些惡意的 SQL 語句，就像是「請你給我第五格資料，並給我全部的密碼」，這時候程式就會以為整句都是正常要求，而進行不該執行的動作，導致駭客成功做到想要做的事情。

當這些惡意的 SQL 語句隨著你的輸入一起傳送給資料庫時，資料庫就會「誤以為」這些惡意的指令是網站程式發出的正常指令，然後就照著執行了！

這種攻擊可能會導致

  • 登入繞過
  • 使用者資料外洩
  • 資料庫被刪除或修改
  • 網站整體被駭客掌控
